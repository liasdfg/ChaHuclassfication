# ChaHuclassfication

    尊敬的师兄您好，目前茶壶的几何和自然形制的分类模型项目已经实现了简单的模型训练与验证，以及测试图片可视化的功能，主要参考了https://github.com/chenyige1/ClayTeapots_Classify这个项目，但是目前还存在不少的问题，还需要不少的优化，这里先简单和您汇报一下。由于github对上传文件大小存在限制，所以数据集和训练出来的模型参数未能上传。

下面我来简单介绍一下classification.py中的函数功能及其实现。

    ChaHuDataSet 是负责整个项目的大部分的数据处理，负责将 HuggingFace Dataset 转换为 PyTorch Dataset。该类首先自动构建类别名称与类别索引之间的映射关系，保证训练和推理阶段标签的一致性；随后利用 mask 提取目标区域，通过寻找 mask 非零区域的边界框对原图进行裁剪，使模型聚焦于茶壶主体而非背景信息。在数据增强阶段采用 Albumentations 对图像和 mask 进行同步变换，避免目标区域与标注错位；最后利用 mask 去除背景并完成 ImageNet 标准化处理，输出符合深度学习模型输入格式的 Tensor 数据。

    unify_label() 用于统一训练集与验证集的标签空间。函数首先统计两个数据集中的类别集合，然后计算共同类别，对于只存在于单侧数据集的类别统一映射为“其他”类别。这样既避免了训练阶段未出现类别导致的验证错误，又保留了原始数据样本数量。该函数主要用于解决训练集与验证集类别分布不一致的问题。

    get_model() 根据输入的模型名称动态构建分类网络，目前支持 ResNet34、GoogLeNet 和 VGG19 三种主流卷积神经网络，并自动替换最后的分类层以适配当前任务的类别数量，并通过加载 ImageNet 预训练权重实现迁移学习，使模型能够利用大规模视觉数据中学习到的通用特征，在小样本场景下可以获得更快的收敛速度和更好的分类性能。

    find_batch_size() 根据硬件资源自动估计合适的 Batch Size，在 GPU 环境下，通过不断扩大 Batch Size 并执行测试，当发生显存溢出时返回上一次成功运行的 Batch Size，可以提高模型训练的自动化程度。

    get_num_workers() 用于自动配置 DataLoader 的并行加载线程数，其根据系统平台和 CPU 物理核心数量动态确定 worker 数量，在 Windows 平台限制较低线程数以保证稳定性，在 Linux 平台则充分利用多核优势提升数据读取速度

    train_model() 实现单轮训练流程，先将模型切换到训练模式，然后依次完成数据加载、前向传播、损失计算、反向传播以及参数更新。对于 GoogLeNet 的多输出结构，额外计算辅助分类器损失并加权融合，从而增强梯度传播效果。训练过程中实时统计 Loss 和 Accuracy，并利用 tqdm 动态显示训练进度，可以兼容多种网络结构。

    validate_model() 实现模型验证流程，先将模型切换到评估模式，并关闭梯度计算以减少显存占用和计算开销，随后在验证集上执行前向推理，统计平均损失和分类准确率，用于衡量模型的泛化能力。

    主程序（Main）负责构建完整的模型训练流水线，定义训练集和验证集的数据增强策略，加载 ChaHu 数据集并完成标签统一、构建数据集对象、分类模型以及 DataLoader，并配置损失函数、优化器和学习率调度器。训练过程中采用 Label Smoothing 提高模型泛化能力，利用 ReduceLROnPlateau 根据验证集损失动态调整学习率，并在验证准确率提升时自动保存最佳模型权重。

    process_image() 是数据预处理函数，负责将原始图片转换为模型可接受的输入格式，函数首先利用 rembg 自动去除背景并生成目标 mask，然后通过计算 mask 的最小外接矩形完成目标裁剪，从而减少背景噪声对分类结果的影响，随后对图像和 mask 同步执行 Albumentations 预处理，并利用 mask 进一步抑制背景区域，最后完成 ImageNet 标准化、维度转换以及 Tensor 化操作，以保证推理阶段与训练阶段的数据处理流程一致。

    主程序（Main）负责完成模型加载、图片预测和结果可视化。首先读取classification.py保存的标签映射文件和最佳模型权重，确保预测类别与训练阶段保持一致，随后遍历指定文件夹中的测试图片，通过 process_image() 完成预处理，并将数据输入训练好的分类模型进行前向传播，获得各类别的预测分数，并利用 Softmax 将模型输出转换为概率分布，计算预测类别及对应置信度。
